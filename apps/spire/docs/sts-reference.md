# 尖塔（spire）数值 / 机制参考

这份文档是 `apps/spire` 复刻杀戮尖塔时的**机制与数值来源指引**。只记功能性规则与事实（数值、结构、出招逻辑），**不含受版权保护的创作文案**（卡面 flavor 斜体、事件叙事、角色台词——那些在代码里一律原创中文）。

## Ground truth 主源

**[gamerpuppy/sts_lightspeed](https://github.com/gamerpuppy/sts_lightspeed)（MIT）** —— RNG 精确复刻的 C++ 模拟器。数值/出招逻辑对齐它，一律取 **ascension 0**（代码里常按 `ascension >= N` 分支，取最低档）。关键文件：

| 内容                           | 文件                                                          |
| ------------------------------ | ------------------------------------------------------------- |
| 敌人出招逻辑 / 意图选择 / 血量 | `src/combat/MonsterSpecific.cpp`、`Monster.cpp`               |
| 敌人伤害表                     | `src/combat/MonsterMoveDamage.cpp`                            |
| 卡效果 / 数值                  | `src/game/Card.cpp`                                           |
| 卡基础伤害表                   | `include/constants/Cards.h`（`cardBaseDamage`）               |
| 血量区间表                     | `include/constants/MonsterIds.h`（`monsterHpRange[id][asc]`） |
| 地图生成                       | `src/game/Map.cpp`                                            |
| 遗物                           | `src/game/Relics.cpp` / `include/constants/Relics.h`          |
| 药水                           | `src/game/Potion.cpp`                                         |
| 事件                           | `src/game/Events.cpp`（**只抄机制/概率/结果，文案原创**）     |

交叉校验：Slay the Spire Wiki（fandom）有人话版出招表 + ascension 修正，读 C++ 吃力时对照。

- **半血分裂(M3d)**：EnemyDef.splitInto，降到 ≤maxHp/2 → currentMove="split"，下回合本体消失、生成分裂体(各 HP=分裂瞬间值)。史莱姆王→大史莱姆→中史莱姆链式。大史莱姆(65-69/64-70)进 strong 池(large_slime 权重2，50/50酸/尖)。
- **灼烧/眩晕/泥泞** 废牌：灼烧回合末在手造成2伤；眩晕带虚无回合末消耗；泥泞占位。
- **Act1 战斗池**（`enemies.ts`，sts_lightspeed asc0）：前 3 场抽 weak 池（邪教徒/颚虫/双红虱/小史莱姆组，各25%；小史莱姆组再50/50两种组成）；其余抽 strong 池（**当前只含已实现怪物子集**：蓝色奴隶主2 : 三红虱2 : 大量史莱姆1；gremlin/red_slaver/looter/large_slime/fungi/exordium 待后续里程碑补齐）。
- **状态/术语** `engine/glossary.ts`：力量/易伤(×1.5)/虚弱(×0.75)/脆弱(格挡×0.75)/仪式/蜷缩/反甲/格挡/能量/消耗/抽牌堆/弃牌堆。
- **伤害结算顺序**：基础+力量 → ×虚弱0.75 → ×易伤1.5 → 向下取整 → 先扣格挡再扣血。
- **触发钩子**（当前手写）：`on_turn_start`(仪式)、`on_attacked`(蜷缩/反甲)。
- **遗物（M3a 轻量地基）** `engine/relics/relics.ts`：state.relics 存 {id,counter}，行为按 id 查表（钩子函数原地改 state）。钩子点 `onCombatStart` / `onCombatEnd`。已实现：燃烧之血(起始·战后回6) · 船锚(战始+10格挡) · 血瓶(战始回2) · 金刚杵(战始+1力量) · 提灯(首回合+1能量) · 弹珠袋(战始全体+1易伤)。获取：宝箱给未持有的普通遗物(兜底金币)；精英/boss 给遗物待 M3b/M3d。**受击反弹/回合始末/出牌计数/亡语等钩子点待后续扩展。**

## 待建系统（本轮目标：四角色完整三幕，无进阶难度）

- **分支地图**（M1）：StS 节点图——每层若干节点，向上连边选路；节点类型 战斗/精英/未知(?)/休息(篝火)/商店/宝箱/Boss；每幕 ~15 层固定结构（首层多入口、第9层宝箱、倒数休息、末层 Boss）。
- **第一幕全内容**（M2）：铁甲战士全卡池(普通/罕见/稀有)、全普通敌人、3 精英(六火妖？地精头目/哨卫/拾荒者)、3 Boss(守卫者/六火之灵/史莱姆王)。
- **遗物系统 + 药水**（M3）：遗物需要一套**事件钩子总线**（战斗开始/回合始末/出牌/受击/获金币/进房间/洗牌/消耗…）；先搭 hook bus 再填遗物数据。药水=战斗中一次性道具。
- **商店 + 事件**（M4）：商店经济(买卡/遗物/药水/移除)；事件只抄机制/概率/结果,**文案原创**。
- **静默/机器人/观者三角色**（M5/M6/M7）：各自起始牌组+起始遗物+专属卡池(不同颜色);机器人有充能球(orb)子系统、观者有姿态(stance)子系统——都是额外机制。
- 通关流程、跨幕状态携带（血量/牌组/遗物/金币/药水）。

## 约束速记

- RNG 必须**可序列化**（xoshiro128\*\*，`engine/rng.ts`）——存档续玩 + 黄金种子回归都压这条。
- 服务自持 JSON 存档 `data/spire/`，**不碰共享 SQLite**。
- agent 侧渲染走 `.hbs`；服务只回结构化数据。顶层工具集固定（渐进式披露），KV 前缀零感知。
- 部署：`pnpm app:deploy spire`（+ agent 若触及）；主仓库部署前 `git ff origin/master` + `pnpm install`。
